# YiRae 相关项目检索与优势分析

检索时间：2026-07-07

说明：本次将“相关旧耕果项目”理解为与 `YiRae` 同类型的照片日记、回忆记录、私人故事本类项目，重点看它们在产品功能和项目展示上的优势。

## YiRae 当前定位

`YiRae` 更偏向“本地化的照片故事本”：

- 支持标题、日期、地点、人物、正文、单图或多图
- 支持收藏、搜索、日历视图
- 支持本地隐私设置与应用锁
- 支持故事分享
- 支持加载网络示例内容

这让它已经具备了“个人回忆相册 + 轻量日记”的核心雏形。

## 相关项目与优势

### 1. StoryPad

链接：

- GitHub: https://github.com/theachoem/storypad
- Google Play: https://play.google.com/store/apps/details?id=com.tc.writestory

优势：

- 时间线表达更强：不是单纯列表，而是强调“连续人生记录”的时间轴体验。
- 编辑能力更丰富：支持加粗、列表、复选框、颜色、字体等丰富排版。
- 回顾能力突出：支持“往年今日”回看，天然适合回忆类产品。
- 组织能力成熟：标签、置顶、收藏、搜索一起配套，信息管理更完整。
- 导出能力强：支持文本、Markdown、完整备份导出，迁移和留存更友好。
- 隐私卖点清晰：本地优先、PIN/FaceID/指纹锁表达得很完整。

对 YiRae 的启发：

- 你这个项目很适合补“往年今日”“标签”“Markdown 导出”这三类高感知功能。
- 现在 YiRae 有图片故事基础，如果把“回顾感”做强，会更有记忆产品辨识度。

来源：

- GitHub README 中列出的 timeline journaling、throwback memories、tags、search、backup、markdown export 等特性
- Google Play 页面中列出的 timeline、throwback、photo memories、organization、privacy、export 特性

### 2. Moodiary

链接：

- GitHub: https://github.com/ZhuJHua/moodiary

优势：

- 平台覆盖广：Android、iOS、Windows、macOS、Linux 都有布局，扩展性强。
- 内容类型丰富：除图片外，还支持音频、视频、绘图。
- 编辑器层次更深：支持 Markdown、纯文本、富文本多种输入方式。
- 数据能力更强：全文搜索、分类、导入导出、WebDAV 备份、局域网同步。
- 智能能力前置：情绪分析、问答、AI 助手、本地 NLP 都是差异化方向。
- 轨迹地图有特色：把“地点”从文本字段提升为可视化生活轨迹。

对 YiRae 的启发：

- 你的“地点”“人物”“时间”字段已经有结构化基础，后续很适合做地图轨迹或分类聚合。
- 如果以后要升级产品层级，优先级很高的是“导出/备份”和“标签分类”。

来源：

- GitHub README 中列出的 cross-platform、multiple editors、multimedia、search and classification、backup and synchronization、trail map、intelligent assistant 等特性

### 3. Easy Diary

链接：

- GitHub: https://github.com/hanjoongcho/aaf-easydiary
- F-Droid: https://f-droid.org/en/packages/me.blog.korn123.easydiary/

优势：

- UX 打磨成熟：项目核心卖点直接就是用户体验优化。
- 基础能力完整：写作、搜索、日历、时间线、锁、主题、字体、图表都比较齐。
- 可定制性很高：支持大量主题和自定义字体，长期使用的个性化体验更强。
- 数据呈现更细：图表让“记录习惯”可视化，这种反馈机制很容易提高留存。
- 分享形式更好：明信片式分享比普通文本分享更有传播感。

对 YiRae 的启发：

- YiRae 已经有照片和故事卡片基础，很适合继续往“回忆卡片海报分享”方向做。
- 如果补上统计图表或月度回顾，会更像一个完整产品，而不只是课程项目。

来源：

- GitHub README 中列出的 search、calendar、timeline、postcards、lock、themes、fonts、charts 等特性
- F-Droid 页面中的 backup and recovery 信息

- 现在项目功能已经有了，但品牌语言还不够集中；如果你要公开展示，这一块很值得补。

## 这些相关项目的共同优势

把这些项目放在一起看，它们的优势主要集中在下面几类：

1. 定位一句话就讲清楚  
   好项目通常会直接告诉用户这是时间线日记、照片日记、隐私日记还是情绪日记。

2. 回顾能力强  
   日历视图、往年今日、时间线、统计图表，都会让“回忆感”变强。

3. 信息组织更成熟  
   标签、分类、搜索、置顶、收藏这类功能能显著提升长期使用体验。

4. 导出和备份完整  
   Markdown 导出、云备份、本地备份、跨端同步，是很多成熟项目的重要加分项。

5. 产品表达比单点功能更完整  
   它们不只是“能记东西”，而是有清楚的使用场景、视觉语言和长期使用逻辑。

## YiRae 相比之下最值得优先补的优势点

如果按投入产出比排序，我建议 `YiRae` 优先补这几项：

1. `README` 加项目截图、架构说明、安装说明  
   这能最快提升 GitHub 展示效果。

2. “往年今日”或“月度回顾”  
   这是回忆类产品最容易出亮点的功能。

3. 标签分类 + 更强搜索  
   让故事内容从“能存”升级为“好找”。

4. Markdown / JSON 导出  
   对开源展示和用户可迁移性都很有帮助。

5. 更完整的分享卡片  
   把现有故事分享能力做成更适合传播的图片卡片或海报。

6. 截图和品牌文案  
   让别人一眼明白 `YiRae` 的核心卖点。

## 一句话结论

`YiRae` 的基础方向是对的，核心优势在“照片 + 故事 + 本地隐私”。  
和同类项目相比，它现在最需要补的不是更多零散功能，而是“回顾能力、组织能力、导出能力、展示表达”这四个层面。
